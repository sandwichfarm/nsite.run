  <action>
    Create `apps/blossom/src/storage/local.ts` implementing `LocalStorageClient` class that mirrors the `StorageClient` interface but uses Deno filesystem APIs instead of Bunny Storage REST API.

    Constructor takes `(baseDir: string, serverUrl: string)`. Set `this.cdnHostname = new URL(serverUrl).host`. Strip trailing slashes from baseDir and serverUrl.

    Method implementations:
    - `put(path, body, _contentType?)`: Ensure parent dir with `Deno.mkdir(dirname(fullPath), { recursive: true })`. Convert body to bytes via `new Response(body).bytes()`. Write with `Deno.writeFile`. Return true.
    - `get(path)`: Try `Deno.readFile`, return `new Response(bytes)`. On error (NotFound), return null.
    - `head(path)`: Try `Deno.stat`, return `new Response(null, { status: 200, headers: { "content-length": String(stat.size) } })`. On error, return null.
    - `delete(path)`: Try `Deno.remove`, return true. On error, return false.
    - `getJson`, `getText`, `getToml`, `putJson`: Delegate to get/put with JSON/text/TOML parsing, matching StorageClient behavior exactly.
    - `blobUrl(sha256)`: `${this.serverUrl}/blobs/${sha256.substring(0,2)}/${sha256}` (NOT CDN URL — local blossom serves directly).
    - `blobPath`, `metaPath`, `listPath`, `reportPath`: Identical logic to StorageClient.
    - `list(prefix)`: Use `Deno.readDir` on `${baseDir}/${prefix}`. Return array of `${prefix}${entry.name}` for files (not directories). Return empty array if directory doesn't exist.

    Import `parse as parseToml` from `@std/toml` and `dirname` from `@std/path` (same imports as StorageClient uses).

    Tests use `Deno.makeTempDir()` for isolation, clean up in afterAll. Test file: `apps/blossom/src/storage/local.test.ts`.

  <action>
    Create `apps/relay/src/dev.ts` that enables the relay to run locally under Deno.

    The file must do the following IN THIS ORDER (ordering is critical — see RESEARCH.md Pitfall 1):

    1. **Set environment variables for local DB** — before main.ts creates the db client:
       ```typescript
       Deno.env.set("BUNNY_DB_URL", "file:./dev-relay.db");
       Deno.env.set("BUNNY_DB_AUTH_TOKEN", "");
       ```

    2. **Inject Bunny.v1.serve polyfill** — synchronously before main.ts is imported:
       ```typescript
       const PORT = parseInt(Deno.env.get("RELAY_PORT") ?? "8081");
       (globalThis as Record<string, unknown>).Bunny = {
         v1: {
           serve: (handler: (req: Request) => Response | Promise<Response>) => {
             Deno.serve({ port: PORT, hostname: "0.0.0.0" }, handler);
           },
         },
       };
       ```

    3. **Patch Request.prototype.upgradeWebSocket** — relay main.ts calls `(request as any).upgradeWebSocket()` which is a Bunny API. Under Deno, patch it to delegate to `Deno.upgradeWebSocket(this)`:
       ```typescript
       (Request.prototype as Record<string, unknown>).upgradeWebSocket = function(this: Request) {
         return Deno.upgradeWebSocket(this);
       };
       ```

    4. **Override the import map for @libsql/client** — relay's `deno.json` maps `@libsql/client/web` to `npm:@libsql/client/web`, which rejects `file:` URLs. The dev entrypoint must use `npm:@libsql/client/node` instead. Since `main.ts` imports from `@libsql/client/web` at the top level, and we need that to resolve to the node client, the cleanest approach is: the env vars above will make `createClient` use `file:./dev-relay.db`. The default Deno exports map for `@libsql/client` routes the `"deno"` condition to the node variant. So we need the import to NOT use the `/web` subpath.

       **Approach:** Since main.ts hardcodes `import { createClient } from "@libsql/client/web"` which goes through the relay deno.json import map, and we cannot change that import map at runtime, the dev entrypoint should NOT import main.ts directly. Instead, duplicate main.ts's ~30 lines of logic using the correct import:
       ```typescript
       import { createClient } from "npm:@libsql/client/node";
       import { initSchema } from "./db.ts";  // db.ts functions are client-agnostic
       import { buildNip11Response } from "./nip11.ts";
       import { handleWebSocketUpgrade } from "./relay.ts";
       ```

       Create the client: `const db = createClient({ url: "file:./dev-relay.db" });`

       Then call `Bunny.v1.serve(handler)` with the same handler logic as main.ts (WebSocket upgrade check, initSchema, NIP-11, fallback response). This is ~25 lines duplicated from main.ts but avoids the import map conflict entirely.

    5. **Log startup**: `console.log(\`[relay] listening on http://localhost:${PORT}\`);`

    Do NOT modify apps/relay/src/main.ts — keep prod code untouched.
    Do NOT add any `if (isDev)` branches to existing files.
