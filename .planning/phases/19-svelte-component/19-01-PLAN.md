<task id="19.1.3">
<title>Refactor LoginModal, NIP46Dialog, and AdvancedConfig to use store context</title>
<read_first>
- packages/deployer/src/components/LoginModal.svelte
- packages/deployer/src/components/NIP46Dialog.svelte
- packages/deployer/src/components/AdvancedConfig.svelte
- packages/deployer/src/lib/store.js
</read_first>
<action>
Three components directly import singleton stores. Refactor them to read stores from Svelte context instead, using the context key `'deployer-stores'`.

**LoginModal.svelte** — currently has `import { session } from '../lib/store.js'`:

Replace with:
```js
import { getContext } from 'svelte';
const { session } = getContext('deployer-stores');
```

Remove the `import { session } from '../lib/store.js'` line. Keep the `import { createEventDispatcher } from 'svelte'` — merge into a single import: `import { createEventDispatcher, getContext } from 'svelte'`.

**NIP46Dialog.svelte** — currently has `import { session } from '../lib/store.js'`:

Replace with:
```js
import { getContext } from 'svelte';
const { session } = getContext('deployer-stores');
```

Remove the `import { session } from '../lib/store.js'` line. Merge `getContext` into existing svelte imports: `import { onMount, tick, createEventDispatcher, getContext } from 'svelte'`.

**AdvancedConfig.svelte** — currently has `import { serverConfig } from '../lib/store.js'`:

Replace with:
```js
import { getContext } from 'svelte';
const { serverConfig } = getContext('deployer-stores');
```

Remove the `import { serverConfig } from '../lib/store.js'` line.

All three components must NOT import from `../lib/store.js` anymore. The `getContext('deployer-stores')` call will return the `{ session, deployState, serverConfig }` object that DeployerWidget.svelte sets via `setContext` in Plan 02.
</action>

- LoginModal.svelte contains `getContext('deployer-stores')`

- AdvancedConfig.svelte contains `getContext('deployer-stores')`
- LoginModal.svelte still imports `createEventDispatcher` from 'svelte'

</acceptance_criteria>
