</read_first>
<action>
Replace the contents of `packages/deployer/src/widget/index.js` with:

```javascript
import { NsiteDeployerElement } from './NsiteDeployerElement.js';

// Register custom element if not already defined (safe for multiple script loads)
if (!customElements.get('nsite-deployer')) {
  customElements.define('nsite-deployer', NsiteDeployerElement);
}

// Export for ESM consumers
export { NsiteDeployerElement };
export const VERSION = '0.1.0';
```

Key differences from stealthis:
- NO auto-inject (stealthis auto-injects a fixed button into document.body). The nsite-deployer element must be explicitly placed by the consumer (D-05).
- Guard `customElements.get()` prevents double-registration if script loaded twice.
- VERSION export preserved for programmatic version checking.
</action>

- File contains `customElements.get('nsite-deployer')` guard
- File contains `export { NsiteDeployerElement }`
- File does NOT contain auto-inject logic (no `document.body.appendChild`)
- File contains `export const VERSION`
</acceptance_criteria>
