```json
{
  "name": "@nsite/deployer",
  "version": "0.1.0",
  "type": "module",
  "private": false,
  "exports": {
    ".": {
      "svelte": "./src/index.js",
      "import": "./dist/deployer.mjs",
      "default": "./dist/deployer.js"
    }
  },
  "main": "dist/deployer.js",
  "module": "dist/deployer.mjs",
  "scripts": {
    "build:widget": "vite build --config vite.widget.config.js",
    "dev:widget": "vite build --config vite.widget.config.js --watch"
  },
  "peerDependencies": {
    "svelte": "^4.0.0"
  },
  "devDependencies": {
    "@sveltejs/vite-plugin-svelte": "^3.1.2",
    "publint": "^0.3.18",
    "vite": "^5.4.0"
  }
}
```
CRITICAL: The `"svelte"` condition MUST come first in the exports object — Node.js and bundlers evaluate conditions in declaration order. The "svelte" condition points at `./src/index.js` (a JS module, NOT a .svelte file) per Pitfall A in research. `@sveltejs/vite-plugin-svelte` and `vite` are devDependencies here so they are available for the widget build; they will be hoisted from root. svelte is a peerDependency (NOT a direct dependency) to avoid bundling a second Svelte copy.
