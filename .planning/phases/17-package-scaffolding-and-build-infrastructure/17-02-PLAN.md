Create `packages/deployer/vite.widget.config.js` — this is the ONLY Vite config in packages/deployer (per D-08, D-09, D-10). The lib source export has NO build step; only the widget bundle needs a Vite config.

```js
import { defineConfig } from 'vite';
import { svelte } from '@sveltejs/vite-plugin-svelte';

export default defineConfig({
  plugins: [
    svelte({
      compilerOptions: {
        css: 'injected'
      }
    })
  ],
  build: {
    lib: {
      entry: 'src/widget/index.js',
      name: 'NsiteDeployer',
      formats: ['iife', 'es'],
      fileName: (format) => format === 'es' ? 'deployer.mjs' : 'deployer.js'
    },
    outDir: 'dist',
    cssCodeSplit: false,
    rollupOptions: {
      output: {
        inlineDynamicImports: true
      }
    }
  }
});
```

Key differences from stealthis reference:
- Adds `svelte()` plugin with `css: 'injected'` (stealthis has no Svelte) per D-10
- Entry is `src/widget/index.js` (not `src/index.ts`) — the widget entry point
- IIFE global name is `NsiteDeployer` (not `@nsite/stealthis`)
- Output filenames: `deployer.mjs` / `deployer.js` per D-03
- `cssCodeSplit: false` ensures all CSS bundles into JS (not split into separate file)
- `inlineDynamicImports: true` ensures single file per format (no chunk splitting)
- NO `external` array — IIFE bundle must be fully self-contained (anti-pattern from research)

After creating the file, run the build from the repo root:
```bash
cd packages/deployer && npx vite build --config vite.widget.config.js
```

Both files should exist and be non-empty. The IIFE file wraps content in an IIFE with `var NsiteDeployer`. The ESM file uses `export`.

**Step 2:** Verify SPA can resolve the @nsite/deployer import through the "svelte" condition:
```bash
cd apps/spa && node -e "
  import { createRequire } from 'module';
  const r = createRequire(import.meta.url);
  const resolved = r.resolve('@nsite/deployer');
  console.log('Resolved to:', resolved);
  if (!resolved.includes('packages/deployer')) {
    console.error('ERROR: Did not resolve to workspace package');
    process.exit(1);
  }
  console.log('RESOLUTION OK');
"
```
Note: Node's `require.resolve` may not honor the "svelte" condition (it uses "default"). The key test is that the import resolves to the workspace package at all (not to a missing package error). The SPA's Vite will use the "svelte" condition at build time — the Node resolution just confirms the workspace symlink is working.
