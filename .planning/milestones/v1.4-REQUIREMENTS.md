# Requirements: nsite.run

**Defined:** 2026-03-24
**Core Value:** Provide reliable, always-available nsite infrastructure that serves sites fast via progressive caching while making the relay and blossom accessible to the broader nsite ecosystem.

## v1.4 Requirements

Requirements for Deploy Safety milestone. Each maps to roadmap phases.

### Deploy Guards

- [x] **GUARD-01**: Root site option shows inline amber warning when user already has a root site deployed
- [x] **GUARD-02**: Root site guard shows existing site info and "Update existing site" button
- [x] **GUARD-03**: Named site dTag field warns inline when entered slug matches an existing named site
- [x] **GUARD-04**: Named site existing-slug warning shows "Update existing site" button to route to update flow
- [x] **GUARD-05**: Deploy waits for site data to load before allowing deploy (sitesLoading race fix)

### Delete Animation

- [x] **DELT-01**: After initiating delete, site card becomes unexpandable with visual "deleting" styling
- [x] **DELT-02**: Site card fades out with animation once deletion is confirmed by relay
- [x] **DELT-03**: If delete fails, card reverts to normal interactive state (rollback)
- [x] **DELT-04**: Remaining site cards reflow smoothly when deleted card exits

### Post-Action Navigation

- [x] **NAV-01**: Deploy/manage tab buttons are disabled (greyed out) while stepper is active
- [x] **NAV-02**: Deploy/manage tab buttons re-enabled after successful operation
- [x] **NAV-03**: "Manage sites" button shown on deploy success screen
- [x] **NAV-04**: "Deploy new site" button shown in manage view

### Leave Confirmation

- [x] **LEAVE-01**: Browser shows leave confirmation when closing/reloading tab during in-progress operation
- [x] **LEAVE-02**: In-page confirmation modal when switching Deploy/Manage tabs during in-progress operation
- [x] **LEAVE-03**: Leave confirmation is not shown when no operation is in progress

## Future Requirements

### Optimization

- **OPT-01**: Pull zone edge rules replace gateway-as-router for cheaper traffic routing
- **OPT-02**: ETag / If-None-Match support for 304 Not Modified responses

### Ecosystem

- **ECO-01**: NIP-05 identity verification for hosted sites
- **ECO-02**: Custom domain support via DNS TXT records

### UX Enhancements

- **UX-01**: Pre-validate dTag against existing named sites at entry time (before deploy button)
- **UX-02**: Deploy history / version rollback via manifest versioning

## Out of Scope

| Feature | Reason |
|---------|--------|
| Typed confirmation ("type OVERWRITE") | Hostile UX for the common update-site workflow |
| Auto-navigate after deploy | Interrupts success state where user needs to copy URL / save nsec |
| Blocking deploy button when site exists | Breaks the update-site workflow entirely |
| Undo delete / soft delete | Decentralized relay+blossom network has no reliable rollback at protocol level |
| Persist deploy state across reload | Binary file data in localStorage is impractical; leave confirmation prevents accidental case |
