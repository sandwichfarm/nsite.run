# Requirements: nsite.run

**Defined:** 2026-03-13
**Core Value:** Provide reliable, always-available nsite infrastructure that serves sites fast via progressive caching while making the relay and blossom accessible to the broader nsite ecosystem.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Infrastructure

- [x] **INFRA-01**: Monorepo build system produces separate bundles for relay, blossom, and gateway Edge Scripts
- [x] **INFRA-02**: Each Edge Script bundle stays under 1MB with CI enforcement (750KB soft warn)
- [x] **INFRA-03**: Shared types package provides common nostr/blossom types across all packages

### Routing

- [x] **ROUTE-01**: Gateway Edge Script routes root domain requests (no npub subdomain) to SPA
- [x] **ROUTE-02**: Gateway Edge Script routes npub subdomain requests to nsite resolver
- [x] **ROUTE-03**: Gateway Edge Script routes WebSocket upgrade requests to relay
- [x] **ROUTE-04**: Gateway Edge Script routes blossom endpoints (/upload, /{sha256}, /list) to blossom server
- [x] **ROUTE-05**: Gateway Edge Script routes named site subdomains (identifier.npub1xxx.nsite.run) to nsite resolver

### Relay

- [x] **RELAY-01**: Relay accepts and stores kind 15128 (root site manifest) events
- [x] **RELAY-02**: Relay accepts and stores kind 35128 (named site manifest) events
- [x] **RELAY-03**: Relay accepts and stores kind 10002 (relay list) events
- [x] **RELAY-04**: Relay accepts and stores kind 10063 (blossom server list) events
- [x] **RELAY-05**: Relay rejects events of any other kind
- [x] **RELAY-06**: Relay implements NIP-01 core protocol (EVENT, REQ, CLOSE, EOSE) over WebSocket
- [x] **RELAY-07**: Relay uses Bunny DB (libSQL) for event storage
- [x] **RELAY-08**: Relay serves NIP-11 relay information document on HTTP GET with appropriate Accept header

### Blossom

- [x] **BLSM-01**: Blossom serves blobs via GET /{sha256} (BUD-01)
- [x] **BLSM-02**: Blossom accepts blob uploads via PUT /upload with kind 24242 Nostr auth (BUD-02)
- [x] **BLSM-03**: Blossom lists blobs by pubkey via GET /list/{pubkey} (BUD-02)
- [x] **BLSM-04**: Blossom deletes blob ownership via DELETE /{sha256} with auth (BUD-02)
- [x] **BLSM-05**: Blossom accepts any blob upload with valid auth (manifest check deferred to v2)
- [x] **BLSM-06**: Blossom uses Bunny Storage for blob persistence
- [x] **BLSM-07**: Blossom validates SHA-256 integrity on upload
- [x] **BLSM-08**: Blossom supports HEAD /{sha256} for existence checks

### Gateway

- [x] **GATE-01**: Gateway resolves npub from subdomain hostname
- [x] **GATE-02**: Gateway resolves named site identifier from subdomain hostname
- [x] **GATE-03**: Gateway fetches user's NIP-65 relay list (kind 10002) to discover outbox relays
- [x] **GATE-04**: Gateway queries manifest events (15128/35128) from user's relays and own relay
- [x] **GATE-05**: Gateway resolves requested path to SHA-256 hash via manifest path tags
- [x] **GATE-06**: Gateway fetches blobs from manifest server tags, then user's 10063 blossom list
- [x] **GATE-07**: Gateway verifies SHA-256 integrity of fetched blobs
- [x] **GATE-08**: Gateway serves files with correct Content-Type (from blossom or path extension)
- [x] **GATE-09**: Gateway falls back to /index.html for directory paths (/ → /index.html)
- [x] **GATE-10**: Gateway serves /404.html for unmatched paths
- [x] **GATE-11**: Gateway handles brotli/gzip compressed assets in manifests
- [x] **GATE-12**: Gateway sets security headers on all responses (CSP, X-Content-Type-Options, etc.)

### Progressive Caching

- [x] **CACHE-01**: Cold cache (never seen) shows loading page, resolves manifest, persists to relay
- [x] **CACHE-02**: Loading page displays user's profile (display name, avatar from kind 0)
- [x] **CACHE-03**: Loading page auto-refreshes until content is available
- [x] **CACHE-04**: Warm outdated cache serves cached site immediately
- [x] **CACHE-05**: Warm cache triggers background update check against user's relays
- [x] **CACHE-06**: When update found, injects banner into served HTML with "click to refresh" link
- [x] **CACHE-07**: Warm current cache serves immediately, background check finds no update
- [x] **CACHE-08**: Gateway persists fetched manifests into its own relay
- [x] **CACHE-09**: Gateway persists fetched blobs into its own blossom

### SPA

- [x] **SPA-01**: SPA serves at root domain (nsite.run, no npub subdomain)
- [x] **SPA-02**: User can authenticate via NIP-07 browser extension
- [x] **SPA-03**: User can authenticate via NIP-46 Nostr Connect / bunker
- [x] **SPA-04**: User can select a folder for upload (directory picker)
- [x] **SPA-05**: User can upload a zip or tar.gz archive
- [x] **SPA-06**: SPA extracts and displays file list for user confirmation before upload
- [x] **SPA-07**: SPA scans files for dangerous filenames (.env, id_rsa, *.pem, etc.) and blocks upload
- [x] **SPA-08**: SPA scans file content with regex patterns for secrets (API keys, tokens) and warns
- [x] **SPA-09**: SPA publishes site manifest (kind 15128 or 35128) to the gateway's relay
- [x] **SPA-10**: SPA uploads file blobs to the gateway's blossom server
- [x] **SPA-11**: SPA includes educational content about nsites (what they are, how they work)
- [x] **SPA-12**: SPA links to other nsite gateways, tools, relays, and blossom servers
- [x] **SPA-13**: SPA is built with Svelte

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Optimization

- **OPT-01**: Pull zone edge rules replace gateway-as-router for cheaper traffic routing
- **OPT-02**: ETag / If-None-Match support for 304 Not Modified responses

### Ecosystem

- **ECO-01**: NIP-05 identity verification for hosted sites
- **ECO-02**: Custom domain support via DNS TXT records

## Out of Scope

| Feature | Reason |
|---------|--------|
| Legacy kind 34128 support | Deprecated format; new manifests only (15128/35128) |
| General-purpose relay | Storage explosion, operational complexity; nsite-only |
| General-purpose blossom | Unbounded storage growth, abuse vector; nsite-only |
| Auto-refresh on update | Disrupts user state; manual banner link instead |
| Payment gating | Contradicts open public infrastructure mission |
| User accounts / subscriptions | Nostr pubkeys are the identity layer |
| Site analytics / tracking | Privacy-violating; owners can embed their own |
| GitHub auto-deploy integration | nsite-action GitHub Action already exists |
| Mobile app | Web SPA sufficient |
| Tor/I2P proxy | Out of scope for v1; niche use case |
