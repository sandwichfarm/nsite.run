**2. Create `apps/relay/src/handler.ts`:**

Three exported async functions implementing NIP-01 message handling:

**`handleEvent(socket: WebSocket, db: Client, event: NostrEvent): Promise<void>`**
Per user's locked decisions on verification and error messages:

a. Validate required fields exist (id, pubkey, sig, kind defined). If missing:
   `["OK", event.id ?? "", false, "invalid: missing fields"]`

b. Validate kind is allowed using `isAllowedKind()` from `@nsite/shared/validation`. If not:
   `["OK", event.id, false, "blocked: kind ${event.kind} not allowed on this relay"]`

c. Check for duplicate using `checkDuplicate(db, event.id)`. If duplicate:
   `["OK", event.id, true, "duplicate: already have this event"]`
   Return early (per locked decision: OK true silently, no message — but NIP-01 OK format requires the message field, so use "duplicate:" prefix).

d. Verify event id: call `getEventHash()` from `@nostr/tools/pure`, compare to event.id. If mismatch:
   `["OK", event.id, false, "invalid: bad event id"]`

e. Verify signature: call `verifyEvent()` from `@nostr/tools/pure`. If false:
   `["OK", event.id, false, "invalid: bad signature"]`

f. Check created_at: reject events with created_at more than 600 seconds (10 min) in the future. OK false with "invalid: event created_at too far in the future".

g. Handle by kind:
   - Kind 5 (deletion): call `deleteEvents(db, event)`. OK true.
   - Kind 10002 or 10063 (replaceable): call `insertReplaceableEvent(db, event)`. OK true.
   - Kind 35128 (parameterized replaceable): call `insertParameterizedReplaceableEvent(db, event)`. OK true.
   - Kind 15128 (regular): call `insertEvent(db, event)`. OK true.

h. After successful storage, check all subscriptions on this connection — for each subscription whose filters match the new event, send `["EVENT", subscriptionId, event]`. Pass the ConnectionState to enable this. Update the function signature to accept ConnectionState as a parameter.

**`handleReq(socket: WebSocket, db: Client, subscriptionId: string, filters: NostrFilter[]): Promise<void>`**

a. Validate subscriptionId is a non-empty string. If invalid, send NOTICE and return.
b. Validate filters array is non-empty and each filter is an object. If invalid, send NOTICE and return.
c. Cap filters count at 10 (per NIP-11 max_filters). If exceeded, send NOTICE "too many filters" and return.
d. Query events using `queryEvents(db, filters)`.
e. For each matched event, send `["EVENT", subscriptionId, event]`.
f. After all events sent, send `["EOSE", subscriptionId]`.
g. Store the subscription in ConnectionState.subscriptions Map (key: subscriptionId, value: { id: subscriptionId, filters }). If subscriptionId already exists, replace it (NIP-01 behavior).

**`handleClose(socket: WebSocket, state: ConnectionState, subscriptionId: string): void`**

a. Remove subscriptionId from state.subscriptions Map.
b. Send `["CLOSED", subscriptionId, ""]` per NIP-01 (optional but good practice). Actually, NIP-01 says relay sends CLOSED only in response to the client's CLOSE — so just remove from the Map. Do NOT send a response message for CLOSE (NIP-01 does not specify one for regular close; CLOSED is for NIP-42 auth). Just silently remove.

Import `getEventHash` and `verifyEvent` from `@nostr/tools/pure`.
Import `isAllowedKind` from `@nsite/shared/validation`.
Import `NsiteKind` from `@nsite/shared/types`.
Import db functions from `./db.ts`.
Import types from `./types.ts`.
Use `import type { Client } from "@libsql/client/web"` for the db client type.

```typescript
import { createDb, initSchema } from "./db.ts";
import { buildNip11Response } from "./nip11.ts";
import { handleWebSocketUpgrade } from "./relay.ts";

// Initialize DB client and schema on module load (cold start)
const db = createDb();
const schemaReady = initSchema(db);

export default {
  async fetch(request: Request): Promise<Response> {
    // Ensure schema is initialized before handling any request
    await schemaReady;

    // NIP-11: HTTP GET with Accept: application/nostr+json
    if (
      request.method === "GET" &&
      request.headers.get("accept")?.includes("application/nostr+json")
    ) {
      return buildNip11Response();
    }

    // WebSocket upgrade
    if (request.headers.get("upgrade")?.toLowerCase() === "websocket") {
      return handleWebSocketUpgrade(request, db);
    }

    // Fallback for plain HTTP requests
    return new Response("nsite relay - connect via WebSocket", {
      status: 200,
      headers: {
        "Content-Type": "text/plain",
        "Access-Control-Allow-Origin": "*",
      },
    });
  },
};
```

NOTE: The fetch handler is now async (it awaits schemaReady). This is correct because there IS an await expression, satisfying the require-await lint rule.
