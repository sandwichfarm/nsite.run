1. Create `apps/relay/src/types.ts` with relay-local types:
   ```typescript
   import type { NostrEvent, NostrFilter } from "@nsite/shared/types";

   /** Parsed client-to-relay messages (NIP-01) */
   export type ClientMessage =
     | ["EVENT", NostrEvent]
     | ["REQ", string, ...NostrFilter[]]
     | ["CLOSE", string];

   /** Relay-to-client messages (NIP-01) */
   export type RelayMessage =
     | ["EVENT", string, NostrEvent]      // subscription_id, event
     | ["OK", string, boolean, string]     // event_id, accepted, message
     | ["EOSE", string]                    // subscription_id
     | ["NOTICE", string];                 // message

   /** Active subscription for a WebSocket connection */
   export interface Subscription {
     id: string;
     filters: NostrFilter[];
   }

   /** Per-connection state managed during WebSocket lifetime */
   export interface ConnectionState {
     subscriptions: Map<string, Subscription>;
     rateLimitBucket: RateLimitBucket;
   }

   /** Rate limit tracking for a single connection */
   export interface RateLimitBucket {
     eventCount: number;
     reqCount: number;
     windowStart: number;
   }
   ```

2. Create `apps/relay/src/schema.ts` with SQL DDL strings. Follow the schema from RESEARCH.md Pattern 7:
   - `events` table with columns: id (TEXT PK), pubkey (TEXT NOT NULL), created_at (INTEGER NOT NULL), kind (INTEGER NOT NULL), tags (TEXT NOT NULL, JSON), content (TEXT NOT NULL), sig (TEXT NOT NULL), raw (TEXT NOT NULL, full JSON), d_tag (TEXT, nullable, for NIP-33), deleted_at (INTEGER, nullable, for soft delete)
   - Indexes: idx_events_pubkey, idx_events_kind, idx_events_created_at (DESC), idx_events_kind_pubkey, idx_events_kind_pubkey_dtag (for NIP-33 lookups)
   - `tags` table with columns: event_id (TEXT NOT NULL REFERENCES events(id) ON DELETE CASCADE), tag_name (TEXT NOT NULL), tag_value (TEXT NOT NULL)
   - Indexes: idx_tags_name_value, idx_tags_event_id
   - `schema_version` table with version INTEGER PRIMARY KEY, INSERT OR IGNORE version 1
   - Export as `const SCHEMA_DDL: string[]` — an array of SQL statements to execute in order.

3. Create `apps/relay/src/db.ts` — the database access layer:
   - Import `createClient` from `@libsql/client/web` and types from `@nsite/shared/types` and local `types.ts`
   - `createDb()`: Returns a libSQL client using env vars `BUNNY_DB_URL` and `BUNNY_DB_AUTH_TOKEN`. If vars are missing, use empty strings (will fail at runtime with clear error).
   - `initSchema(db)`: Runs each DDL statement from schema.ts via `db.execute()`. Called once on startup. Idempotent (CREATE TABLE IF NOT EXISTS).
   - `insertEvent(db, event: NostrEvent)`: Insert event into events table + insert all single-letter tags into tags table. Use `db.batch([...], "write")` for atomicity. Extract d_tag from event.tags for NIP-33 kinds. Store `raw` as `JSON.stringify(event)`.
   - `insertReplaceableEvent(db, event: NostrEvent)`: For kinds 10002, 10063 (NIP-01 replaceable): DELETE existing where kind=? AND pubkey=? AND created_at<=?, then INSERT OR IGNORE. Use batch for atomicity.
   - `insertParameterizedReplaceableEvent(db, event: NostrEvent)`: For kind 35128 (NIP-33): Extract d_tag, DELETE existing where kind=? AND pubkey=? AND d_tag=? AND created_at<=?, then INSERT OR IGNORE + tag inserts. Use batch.
   - `queryEvents(db, filters: NostrFilter[])`: Build SQL query from filter fields (ids, authors, kinds, since, until, limit, single-letter tag filters like #d, #e, #p). Use the query building pattern from RESEARCH.md. Support multiple filters by running each and deduplicating results by event id. Return NostrEvent[] parsed from the `raw` column. Cap limit at 500 per filter.
   - `deleteEvents(db, deletionEvent: NostrEvent)`: NIP-09 handler. Extract referenced event ids from `e` tags in the deletion event. For each referenced id, verify the referenced event has the same pubkey as the deletion event (same author). Soft-delete by setting `deleted_at = deletion_event.created_at` on matching events. Then insert the deletion event itself.
   - `checkDuplicate(db, eventId: string)`: Returns true if event id already exists. Used before full insert to short-circuit with OK true.

   Use parameterized queries (?) for all user-supplied values. Never interpolate strings into SQL.
