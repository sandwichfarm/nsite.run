<task type="auto">
  <name>Task 2: Live resolver with behavioral tests — resolution pipeline, cache state machine, file serving</name>
  <files>apps/gateway/src/resolver.ts, apps/gateway/src/resolver.test.ts</files>
  <action>
    Create `apps/gateway/src/resolver.ts` — this is the largest and most critical module in the project. It replaces `handleResolverStub` and handles ALL nsite subdomain requests.

    **Export: `async function handleResolver(request: Request, pointer: SitePointer): Promise<Response>`**

    **Module-level initialization (runs once per edge worker):**
    ```typescript
    const db = createDb();
    const storage = createStorageClient(); // null if env vars missing
    const SEED_RELAYS = (Deno.env.get("SEED_RELAYS") || "wss://purplepag.es,wss://relay.damus.io,wss://relay.nostr.band").split(",").map(s => s.trim());
    ```
    Call `initSchema(db)` lazily on first request (use a module-level `let schemaReady = false` flag).

    **Request flow:**

    1. **Decode npub** — `npubToHex(pointer.npub)`. If null, return 400 "Invalid npub".

    2. **Compute cache key** — `cacheKey(pubkeyHex, pointer.identifier)`.

    3. **Handle `/_nsite/ready` polling endpoint** — If `url.pathname === "/_nsite/ready"`, read the `k` query param, look up in siteCache. Return `{"ready": true}` if state is "ready" or "not-found", `{"ready": false}` if "loading" or not in cache. Content-Type: application/json. This endpoint is called by the loading page JS polling.

    4. **Check cache:**
       - **Cache hit, state "ready"** → serve file (step 7). Trigger background update check (step 6).
       - **Cache hit, state "loading"** → someone else is already resolving. Return loading page for HTML-like paths, 404 for asset paths.
       - **Cache hit, state "not-found"** → return not-found page. Optionally trigger background re-check if cached for >5 minutes.
       - **Cache miss** → create CacheEntry in "loading" state, start resolution (step 5).

    5. **Resolution pipeline (streaming/opportunistic)** — Per CONTEXT.md locked decision: parallel, not sequential. Each piece of data triggers the next hop immediately.

       **Step 5a: Parallel initial queries** — Fire all simultaneously:
       - Own relay DB query for manifest (kind 15128 if root, 35128 with d-tag filter for named site)
       - Own relay DB query for kind 10002 (relay list) by pubkey
       - Own relay DB query for kind 10063 (blossom list) by pubkey
       - Seed relay queries for kind 10002 by pubkey (via `queryMultipleRelays`)
       - Seed relay queries for kind 0 (profile, for loading page display name/avatar)

       **Step 5b: As relay list events arrive** — Extract relay URLs via `getRelayUrls()`. Immediately fire manifest queries to those relays via `queryMultipleRelays`:
       - For root site: filter `{ kinds: [15128], authors: [pubkeyHex], limit: 1 }`
       - For named site: filter `{ kinds: [35128], authors: [pubkeyHex], "#d": [identifier], limit: 1 }`

       **Step 5c: Collect manifest** — Use `Promise.race` / `Promise.allSettled` across all manifest sources (own DB + external). Pick the manifest with the highest `created_at`. If no manifest found within total timeout (30s), set state to "not-found".

       **Step 5d: Parse manifest** — `getManifestFiles(manifest)` for path-to-sha256 map. `getManifestServers(manifest)` for blossom server list. Also query kind 10063 (blossom list) from own DB and external relays for additional servers.

       **Step 5e: Update cache entry** — Set `manifestEvent`, `files`, `blossomServers`, `cachedAt = Date.now()`, `state = "ready"`.

       **Step 5f: Persist to own relay (CACHE-08)** — Call `insertReplaceableEvent(db, manifest)` for kind 15128 or `insertParameterizedReplaceableEvent(db, manifest)` for kind 35128. Fire-and-forget (don't block response).

       **Step 5g: Serve the requested file** (same as step 7 below, but at the end of cold-cache resolution).

    6. **Background update check (warm cache)** — Per CACHE-05:
       - Check if a background check is already running for this key: `if (backgroundChecks.has(key)) return;`
       - Create the Promise SYNCHRONOUSLY before any await (Pitfall 1 — race condition prevention):
         ```typescript
         const promise = (async () => { ... })();
         backgroundChecks.set(key, promise);
         ```
       - Inside the async: query manifest from own DB + seed relays + user's relays (same pipeline as step 5, but lighter — skip profile fetch).
       - Compare result: if `fetchedEvent.id !== cachedEntry.manifestEvent.id && fetchedEvent.created_at > cachedEntry.manifestEvent.created_at`, update the cache entry with the new manifest. Per Pitfall 4 — both conditions must be true.
       - Clean up: `backgroundChecks.delete(key)` when done.
       - The CURRENT request serves from old cache. The NEXT request will get the update banner (or the updated content if same request checks after update).

    7. **File serving** — Given a cache entry in "ready" state and a request pathname:
       a. `resolveIndexPath(pathname)` — normalize directory paths.
       b. Look up normalized path in `entry.files` Map. Handle compression: if not found, try path + ".br" and path + ".gz" variants in the files Map. If a compressed variant is found, note the Content-Encoding.
       c. If path not found: check for `/404.html` in files Map → fetch and serve. If no `/404.html`, serve `renderDefault404()`.
       d. Get sha256 from files Map.
       e. **Fetch blob**: Try own blossom first (if storage client available): `storage.get(storage.blobPath(sha256))`. If not found, try each blossom server from `entry.blossomServers` in order: `fetch(\`\${serverUrl.replace(/\\/$/, "")}/${sha256}\`)`. Per Pitfall 7 — strip trailing slash, append sha256.
       f. **Verify SHA-256 (GATE-07)**: Read blob as ArrayBuffer, compute `sha256Hex(new Uint8Array(arrayBuffer))`, compare against expected hash. If mismatch, try next server. If all fail, return 502.
       g. **Persist blob to own blossom (CACHE-09)**: `storage.put(storage.blobPath(sha256), arrayBuffer, contentType)`. Fire-and-forget.
       h. **Detect Content-Type (GATE-08)**: `detectContentType(resolvedPath)` from content-type.ts.
       i. **Detect compression (GATE-11)**: If file was fetched from a compressed path variant, set Content-Encoding header.
       j. **Banner injection (CACHE-06)**: If cache state is warm-outdated (background check found update), AND response Content-Type is text/html, call `injectBanner(htmlString)`. Per Pitfall 3 — ONLY inject into text/html.
       k. **Security headers (GATE-12)**: Spread `securityHeaders()` into response headers.
       l. Return Response with body, status 200, correct Content-Type, Content-Encoding (if compressed), security headers.

    **Loading page for cold-cache HTML requests (Pitfall 2 guard):**
    - Only show loading page for paths that are likely HTML: `pathname === "/" || pathname.endsWith(".html") || !pathname.includes(".")`.
    - For non-HTML paths (has extension like .css, .js, .png): return 404 immediately during cold cache. These assets will be requested again by the browser once the page loads.
    - Fetch profile data (kind 0) for the loading page: if available from the parallel seed relay query, parse `content` as JSON for `{ name, display_name, picture }`. Use `escapeHtml()` on all values.
    - Call `renderLoadingPage({ displayName, avatarUrl, cacheKey })`.

    **Timeouts (per RESEARCH.md recommendations):**
    - Own relay DB query: 2000ms
    - External relay WebSocket: 3000ms per relay
    - Seed relay queries total: 5000ms
    - Manifest queries total: 8000ms
    - Blob fetch per server: 10000ms
    - Total resolution: 30000ms hard limit → return 504 Gateway Timeout

    **Error handling:**
    - All external fetches wrapped in try/catch — failures resolve to empty/null, never throw.
    - Blob fetch failures: try next server in list. All servers fail → 502 Bad Gateway.
    - Total timeout → 504 Gateway Timeout.
