    - detectContentType(".woff2") returns "font/woff2"

    - detectContentType("") returns "application/octet-stream"
    - resolveIndexPath("/") returns "/index.html"

    - detectCompression("/style.css.br") returns { encoding: "br", basePath: "/style.css" }
    - detectCompression("/app.js.gz") returns { encoding: "gzip", basePath: "/app.js" }
    - detectCompression("/index.html") returns null
  </behavior>
  <action>
    Create `apps/gateway/src/content-type.ts` with three exported functions:

    1. `detectContentType(path: string): string` — Extract extension from path using `path.substring(path.lastIndexOf(".")).toLowerCase()`. Look up in a static `CONTENT_TYPES` record covering: .html, .css, .js, .mjs, .json, .svg, .png, .jpg, .jpeg, .gif, .webp, .avif, .ico, .woff, .woff2, .ttf, .otf, .xml, .txt, .md, .wasm, .pdf, .mp3, .mp4, .webm, .ogg, .zip, .gz. Return "application/octet-stream" for unknown extensions.

    2. `resolveIndexPath(pathname: string): string` — If pathname is "/" return "/index.html". If pathname ends with "/" return pathname + "index.html". Otherwise return pathname unchanged. Per CONTEXT.md locked decision.

    3. `detectCompression(path: string): { encoding: string; basePath: string } | null` — If path ends with ".br" return { encoding: "br", basePath: path.slice(0, -3) }. If path ends with ".gz" return { encoding: "gzip", basePath: path.slice(0, -3) }. Otherwise return null.

    Write tests first in `apps/gateway/src/content-type.test.ts` covering all behaviors above.

    Update `apps/gateway/build.ts` to add the esbuild `loader` option: `loader: { ".html": "text" }`. This tells esbuild to inline the `.html` file content as a string constant at bundle time. This satisfies both: (a) the user's decision for a static HTML file, and (b) Bunny Edge's no-filesystem constraint (Pitfall 8) — at runtime the template is a bundled string, but the source is a real `.html` file that can be edited with HTML tooling, syntax highlighting, etc.

Choose whichever approach works with both `deno test` and `esbuild`. If the import assertion approach doesn't work with Deno, fall back to `Deno.readTextFileSync` with `import.meta.url` resolution — this works in Deno tests and esbuild replaces it with the inlined text at bundle time via the `.html: text` loader.
