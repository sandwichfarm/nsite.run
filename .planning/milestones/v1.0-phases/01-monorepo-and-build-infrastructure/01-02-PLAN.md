  <name>Task 1: Create esbuild build scripts for all three edge scripts</name>
  <files>
    apps/relay/build.ts,
    apps/blossom/build.ts,
    apps/gateway/build.ts
  </files>
  <action>
    Create a `build.ts` in each app directory (relay, blossom, gateway). Each script follows the same pattern from the research document:

    ```typescript
    import * as esbuild from "npm:esbuild@^0.27.3";
    import { denoPlugins } from "jsr:@luca/esbuild-deno-loader@^0.11.1";
    import { builtinModules } from "node:module";
    ```

    Key esbuild configuration for each:
    - `plugins: [...denoPlugins({ configPath: new URL("./deno.json", import.meta.url).pathname })]`
      - CRITICAL: `configPath` MUST be absolute. Use `new URL("./deno.json", import.meta.url).pathname` to resolve relative to the build script, NOT relative to cwd.
    - `entryPoints: ["./src/main.ts"]`
    - `outfile`: `"./dist/{name}.bundle.js"` where name is relay, blossom, or gateway
    - `bundle: true`
    - `format: "esm"`
    - `minify: true`
    - `external`: all Node.js built-in modules — `[...builtinModules, ...builtinModules.map((m) => "node:" + m)]`
    - `banner.js`: Polyfill banner for Bunny runtime compatibility:
      ```
      import * as process from "node:process";import { Buffer } from "node:buffer";globalThis.process ??= process;globalThis.Buffer ??= Buffer;globalThis.global ??= globalThis;
      ```
    - `define: { "process.env.NODE_ENV": '"production"' }`
    - `target: "esnext"` (Bunny runtime supports modern JS)

    After `esbuild.build()`, ALWAYS call `await esbuild.stop()` — Deno will hang otherwise because esbuild starts a child process.

    Log the output file path and size after build completes.

    **IMPORTANT:** If `denoPlugins` with the member-level `configPath` fails to resolve `@nsite/shared` workspace imports, try pointing `configPath` at the root workspace `deno.json` instead: `new URL("../../deno.json", import.meta.url).pathname`. Test both approaches — the research flagged this as an open question (Open Question 1).

    Ensure each app's `dist/` directory is created by the build script if it doesn't exist — use `await Deno.mkdir("./dist", { recursive: true })` before the esbuild call.
  </action>
  <verify>
    <automated>cd /home/sandwich/Develop/nsite.run && deno task build && ls -la apps/relay/dist/relay.bundle.js apps/blossom/dist/blossom.bundle.js apps/gateway/dist/gateway.bundle.js</automated>
  </verify>
  <done>All three build scripts produce bundle files. `deno task build` runs all three builds via `deno task --recursive build`. Each bundle is a valid ESM JavaScript file. @nsite/shared imports are resolved and inlined into the bundles.</done>

<task type="auto">
  <name>Task 2: Create bundle size enforcement script</name>
  <files>scripts/check-bundle-sizes.ts</files>
  <action>
    Create `scripts/check-bundle-sizes.ts` that enforces bundle size limits:

    - Import `BUNDLE_SIZE_LIMIT` and `BUNDLE_SIZE_WARN` from `@nsite/shared/constants` (use the constants defined in Plan 01 — 1_000_000 and 750_000 respectively).
    - Define the three bundle paths:
      - `apps/relay/dist/relay.bundle.js`
      - `apps/blossom/dist/blossom.bundle.js`
      - `apps/gateway/dist/gateway.bundle.js`
    - For each bundle:
      - Read file size via `Deno.stat(path)`
      - If size > BUNDLE_SIZE_LIMIT (1MB): print `FAIL: {name} is {kb}KB -- exceeds 1MB hard limit` to stderr, set failed flag
      - If size > BUNDLE_SIZE_WARN (750KB): print `WARN: {name} is {kb}KB -- approaching 1MB limit (warn at 750KB)` to stderr
      - Otherwise: print `OK: {name} is {kb}KB` to stdout
    - Also output a JSON summary line for CI consumption: `{ "relay": { "size": N, "status": "ok|warn|fail" }, ... }`
    - If any bundle failed: `Deno.exit(1)`
    - If all passed (even with warnings): exit 0

    Handle missing bundle files gracefully — if a bundle doesn't exist, report as FAIL with message "bundle not found (did you run deno task build?)".

    After creating the script, run the full pipeline: `deno task build` then `deno run --allow-read scripts/check-bundle-sizes.ts` to verify end-to-end.
  </action>
  <verify>
    <automated>cd /home/sandwich/Develop/nsite.run && deno task build && deno run --allow-read scripts/check-bundle-sizes.ts</automated>
  </verify>
  <done>Bundle size script reports sizes for all 3 bundles, exits 0 when all are under 1MB. Script correctly imports constants from @nsite/shared. JSON summary line is output for CI parsing. Missing bundle files produce FAIL status.</done>
