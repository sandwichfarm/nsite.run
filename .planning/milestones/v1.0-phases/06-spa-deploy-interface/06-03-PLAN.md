  <action>
1. **src/lib/store.js** — Svelte store with localStorage persistence:
   - `persistedStore(key, defaultValue)` — writable store that reads initial value from localStorage (JSON.parse) and subscribes to write changes back (JSON.stringify). Handle JSON parse errors gracefully (fall back to default).
   - `session` — persistedStore('nsite_session', { pubkey: null, signerType: null, displayName: null, avatar: null, npub: null }).
   - `deployState` — writable store (NOT persisted): { step: 'idle', files: [], warnings: [], progress: 0, result: null, error: null }.
   - `serverConfig` — persistedStore('nsite_servers', { extraRelays: [], extraBlossoms: [] }).

2. **src/lib/nostr.js** — Nostr identity and communication:
   - `DEFAULT_RELAYS` — ['wss://relay.damus.io', 'wss://relay.primal.net'] (for NIP-46 + bootstrap queries).
   - `NSITE_RELAY` — 'wss://nsite.run'.
   - `NSITE_BLOSSOM` — 'https://nsite.run'.
   - `getRelayPool()` — Lazy singleton RelayPool from applesauce-relay.
   - `createAnonymousSigner()` — `new PrivateKeySigner()`, return `{ signer, pubkey: await signer.getPublicKey(), nsec: nsecEncode(signer.key) }`. Use nostr-tools nip19.nsecEncode for nsec.
   - `createExtensionSigner()` — `new ExtensionSigner()`, try getPublicKey(), catch ExtensionMissingError. Return `{ signer, pubkey }`.
   - `createNostrConnectSigner(relays)` — Create PrivateKeySigner for client keypair. Set NostrConnectSigner.pool. Create NostrConnectSigner with relays (use DEFAULT_RELAYS, NOT wss://nsite.run — nsite relay rejects kind 24133). Get URI via getNostrConnectURI({name: 'nsite.run Deploy', url: 'https://nsite.run'}). Return `{ signer, uri, waitForSigner: () => signer.waitForSigner() }`.
   - `connectFromBunkerURI(bunkerUri)` — `NostrConnectSigner.fromBunkerURI(bunkerUri, {})`. Return `{ signer, pubkey }`.
   - `fetchProfile(pubkey, relays)` — Query kind 0 filter {kinds: [0], authors: [pubkey], limit: 1} from relays. Parse JSON content for {name, display_name, picture, about}. Return parsed profile or null.
   - `fetchRelayList(pubkey, relays)` — Query kind 10002 filter. Parse 'r' tags into relay URLs. Return string[].
   - `fetchBlossomList(pubkey, relays)` — Query kind 10063 filter. Parse 'server' tags. Return string[].

   For fetchProfile/fetchRelayList/fetchBlossomList: Use applesauce-relay pool.relay(url).subscribe() or a simple WebSocket fetch pattern. Since these are one-shot queries, implement a simple `queryRelay(url, filter)` helper that opens a WebSocket, sends REQ, collects events until EOSE, closes.

   NOTE: store.js and nostr.js depend on browser APIs (localStorage, applesauce-signers, WebSocket). Unit tests are impractical without mocking the entire runtime environment. These modules are verified via build success here and tested manually via the UI in Plan 04/05. The upload/publish tests in Task 2 cover the pure-logic integration points.

  <action>
1. **src/lib/upload.js** — BUD-02 blob upload:
   - `buildAuthEvent(sha256hex, verb = 'upload')` — Return event template (NOT signed): `{ kind: 24242, created_at: Math.floor(Date.now()/1000), tags: [['t', verb], ['x', sha256hex], ['expiration', String(Math.floor(Date.now()/1000) + 3600)]], content: 'Upload blob via nsite.run' }`.
   - `uploadBlob(fileData, sha256hex, signer, blossomUrl)` — HEAD check first. If 200, return {skipped: true}. Otherwise, build auth event, sign with signer.signEvent(), encode as `Nostr ${btoa(JSON.stringify(signedEvent))}`. PUT to `${blossomUrl}/upload` with Authorization header and Content-Type 'application/octet-stream'. Return {skipped: false, status}.
   - `uploadAllBlobs(files, signer, blossomUrls, onProgress)` — For each blossom URL, upload each file. Call onProgress({uploaded, total, current}) after each. Use sequential upload (not parallel) to avoid overwhelming the server. Return {uploaded: number, skipped: number, failed: [{path, error}]}.

2. **src/lib/publish.js** — Manifest building and relay publishing:
   - `buildManifest(files, servers, spaFallback)` — Build kind 15128 event template. Files is `[{path, sha256}]`. Create path tags `['path', path, sha256]` for each file. Add `['server', url]` for each server. If spaFallback is true AND files includes a '/index.html', add `['path', '/404.html', indexSha256]`. Return unsigned event template.
   - `publishToRelay(signedEvent, relayUrl)` — Open WebSocket to relayUrl. Send `JSON.stringify(['EVENT', signedEvent])`. Wait for `['OK', eventId, true, ...]` response. Timeout after 10 seconds. Return {success: boolean, message?}.
   - `publishManifest(signer, files, servers, relays, spaFallback)` — Build manifest via buildManifest, sign via signer.signEvent(), publish to each relay via publishToRelay. Return {event, results: [{relay, success, message}]}.

3. **src/lib/__tests__/upload.test.js**:
   - Test `buildAuthEvent` returns correct structure: kind 24242, has t/x/expiration tags, expiration is ~3600s in future.
   - Test `buildAuthEvent` with different verbs (upload, delete).
   - Test auth header format construction: `Nostr ${btoa(JSON.stringify(event))}` (test the encoding logic separately if possible).

4. **src/lib/__tests__/publish.test.js**:
   - Test `buildManifest` with a list of files — verify kind 15128, path tags match input, server tags present.
   - Test `buildManifest` with spaFallback=true — verify /404.html tag exists with correct sha256.
   - Test `buildManifest` with spaFallback=true but no index.html — verify /404.html tag is NOT added.
   - Test `buildManifest` with spaFallback=false — verify no /404.html tag.
   - Test `buildManifest` with multiple servers — verify each gets a server tag.

Note: uploadBlob and publishToRelay involve network/WebSocket — test only the pure builder functions (buildAuthEvent, buildManifest). The network functions are integration-tested via manual UI testing.
