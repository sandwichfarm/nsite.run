  <action>
1. **apps/gateway/src/stubs/spa.ts** — Replace stub with real SPA handler:
   - Rename export from `handleSpaStub` to `handleSpa`.
   - Read `SPA_ASSETS_URL` from `Deno.env.get("SPA_ASSETS_URL")` — this is the Bunny Storage or CDN base URL where SPA dist files are hosted (e.g., `https://nsite-spa.b-cdn.net`).
   - On request to root domain:
     a. Parse pathname from request URL. Default '/' to '/index.html'.
     b. If SPA_ASSETS_URL is set: fetch `${SPA_ASSETS_URL}${pathname}` and return the response with correct Content-Type. Cache-Control: `public, max-age=3600` for assets, `no-cache` for index.html.
     c. If SPA_ASSETS_URL is not set: fall back to a simple HTML response with a message "nsite.run SPA not yet deployed" (graceful degradation during initial setup).
   - For SPA client-side routing: if the fetched path returns 404 from storage, serve index.html instead (SPA fallback at gateway level).
   - Add security headers (X-Content-Type-Options, X-Frame-Options) same as resolver responses.

2. **apps/gateway/src/router.ts** — Update import:
   - Change `import { handleSpaStub } from "./stubs/spa.ts"` to `import { handleSpa } from "./stubs/spa.ts"`.
   - Change line 118 from `return handleSpaStub(request)` to `return handleSpa(request)`.
   - Note: `handleSpa` is now async (fetch to storage), but `route()` is already async, so no signature change needed.

3. **Update .github/workflows/ci.yml** — Add SPA build step:
   - After existing build steps, add:
   ```yaml
   - name: Build SPA
     run: |
       cd apps/spa
       npm ci
       npm run build
   ```
   - This ensures CI validates the SPA builds successfully.

4. **Update .github/workflows/deploy.yml** — Add SPA deploy step:
   - After building, add step to upload `apps/spa/dist/` to Bunny Storage:
   ```yaml
   - name: Build SPA
     run: |
       cd apps/spa
       npm ci
       npm run build
   - name: Deploy SPA to Bunny Storage
     run: |
       cd apps/spa/dist
       for file in $(find . -type f); do
         path="${file#./}"
         curl -s -X PUT \
           -H "AccessKey: ${{ secrets.BUNNY_STORAGE_PASSWORD }}" \
           -H "Content-Type: $(file --mime-type -b "$file")" \
           --data-binary "@$file" \
           "https://${{ secrets.BUNNY_STORAGE_HOSTNAME }}/${{ secrets.BUNNY_STORAGE_USERNAME }}/nsite-spa/$path"
       done
   ```
   - Alternative: If Bunny CLI is available, use that instead. The curl approach works universally.

5. **Verify gateway tests still pass** — Run `deno test --allow-all apps/gateway/` to ensure the import rename doesn't break existing router tests. Update router.test.ts if it references `handleSpaStub` — change to `handleSpa`.

6. **Verify gateway build** — Run `cd apps/gateway && deno task build` to ensure bundle still builds. Check that bundle size doesn't increase significantly (handleSpa is a thin proxy, not inlining SPA assets).
