    - extractZip(ArrayBuffer) returns array of {path, data, size} objects, skipping directory entries
    - extractZip strips common root directory prefix (e.g., "dist/index.html" -> "/index.html")
    - extractTarGz(ArrayBuffer) returns array of {path, data, size} filtering to type === 'file'
    - buildFileTree(files[]) returns nested tree structure {name, path, children?, size?, isDir}
    - Files with paths starting with .git/, .svn/, .hg/ are filtered out by autoExcludeVCS()
    - pickDirectory and pickArchive are async functions (browser-API dependent, not unit-testable — test extractors only)
  </behavior>
  <action>
1. **src/lib/crypto.js** — Export `hashFile(arrayBuffer)`:
   ```js
   export async function hashFile(arrayBuffer) {
     const hashBuffer = await crypto.subtle.digest('SHA-256', arrayBuffer);
     return Array.from(new Uint8Array(hashBuffer))
       .map(b => b.toString(16).padStart(2, '0'))
       .join('');
   }
   ```

2. **src/lib/files.js** — Export:
   - `extractZip(arrayBuffer)` — Use fflate `unzip`. Convert `{ 'path': Uint8Array }` to `[{ path: '/normalized', data: Uint8Array, size: number }]`. Skip entries ending with '/'. Detect and strip common root prefix (if all files share same top-level dir).
   - `extractTarGz(arrayBuffer)` — Use fflate `gunzip` then nanotar `parseTar`. Filter `type === 'file'` or `type === undefined` (nanotar omits type for files). Normalize paths with leading '/'.
   - `autoExcludeVCS(files)` — Filter out files whose paths start with `/.git/`, `/.svn/`, `/.hg/`, `/.DS_Store`. Return `{ included, excluded, excludedCount }`.
   - `buildFileTree(files)` — Convert flat file list `[{path, size}]` into nested tree `{name, path, children, size, isDir}` for tree component rendering. Sort directories first, then files alphabetically.
   - `pickDirectory()` — Feature-detect `showDirectoryPicker`. If available, call it and recursively read via `readDirectoryHandle`. Otherwise, create and click a hidden `<input type="file" webkitdirectory multiple>`. Return `[{path, data, size}]`.
   - `pickArchive()` — Create and click a hidden `<input type="file" accept=".zip,.tar.gz,.tgz">`. Based on extension, call `extractZip` or `extractTarGz`. Return `[{path, data, size}]`.
   - `readDirectoryHandle(dirHandle, basePath)` — Recursive async generator/accumulator for File System Access API.

3. **src/lib/__tests__/files.test.js** — Tests using vitest:
   - Test `extractZip` with a programmatically created ZIP (use fflate `zip` to create test fixture in setup).
   - Test `extractTarGz` — since creating tar.gz in JS is harder, test with a known small fixture or mock parseTar.
   - Test `buildFileTree` with flat file list input, verify nested structure.
   - Test `autoExcludeVCS` filters .git/ paths and returns correct counts.
   - Do NOT test pickDirectory/pickArchive (browser API dependent).

Note: For ZIP test fixtures, use fflate's `zip` function in the test to create a buffer, then extract it with `extractZip`. This proves the round-trip works.
