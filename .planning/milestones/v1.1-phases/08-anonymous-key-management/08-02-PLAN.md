<objective>
Create a logout confirmation modal that prevents anonymous users from accidentally losing their private key. The modal requires explicit backup acknowledgment before allowing logout.

Purpose: Without this safeguard, an anonymous user can click Logout and permanently lose access to their deployed site. The modal forces them to consciously choose to download or copy their nsec before proceeding.

Output: New LogoutConfirmModal.svelte component, updated Navbar.svelte that intercepts anonymous logout.
</objective>

    Create a new file `apps/spa/src/components/LogoutConfirmModal.svelte` following the established modal pattern from LoginModal.svelte.

    **Props:**
    - `export let show = false;` — controls modal visibility
    - `export let nsec = '';` — the nsec1... string to back up
    - `export let npub = '';` — the npub for filename generation

    **Events (via createEventDispatcher):**
    - `dispatch('close')` — user cancelled / closed modal
    - `dispatch('confirm')` — user confirmed logout after backup acknowledgment

    **Internal state:**
    - `let backedUp = false;` — checkbox state for "I've backed up my key"
    - `let nsecCopied = false;` — clipboard copy feedback state

    **Script block:**
    ```javascript
    import { createEventDispatcher } from 'svelte';
    import { downloadNsecFile } from '../lib/nostr.js';

    export let show = false;
    export let nsec = '';
    export let npub = '';

    const dispatch = createEventDispatcher();

    let backedUp = false;
    let nsecCopied = false;

    async function copyNsec() {
      if (!nsec) return;
      await navigator.clipboard.writeText(nsec);
      nsecCopied = true;
      setTimeout(() => (nsecCopied = false), 2000);
    }

    function handleDownload() {
      downloadNsecFile(nsec, npub);
    }

    function handleConfirm() {
      dispatch('confirm');
      backedUp = false;
    }

    function closeModal() {
      backedUp = false;
      nsecCopied = false;
      dispatch('close');
    }
    ```

    **Template structure** (inside `{#if show}` guard):
    1. Fixed backdrop with `bg-black/70`, role="dialog", aria-modal="true", aria-label="Logout confirmation"
    2. Click on backdrop calls closeModal. Escape key calls closeModal.
    3. Add `<!-- svelte-ignore a11y-click-events-have-key-events a11y-no-noninteractive-element-interactions -->` before the backdrop div (matches LoginModal pattern).
    4. Inner container: `bg-slate-800 rounded-xl shadow-2xl w-full max-w-md`
    5. Content padding `p-6`:

    **Header:**
    - Warning icon (amber, same SVG as SuccessPanel's warning triangle)
    - Title: "Back up your key before logging out"
    - Close X button (top-right, matching LoginModal)

    **Warning message:**
    - `<p class="text-amber-300 text-sm mb-4">You deployed with an anonymous identity. If you log out without saving your private key, you will permanently lose access to your site.</p>`

    **nsec display + action buttons:**
    - Amber-bordered box showing the nsec (same styling as SuccessPanel)
    - `<code>` element with nsec, break-all, amber-200 text
    - Two buttons below the nsec display, in a flex row with gap-2:
      - Download button (primary, purple-600): calls handleDownload
      - Copy button (secondary, amber-700): calls copyNsec, shows "Copied!" feedback

    **Checkbox:**
    - `<label class="flex items-center gap-3 cursor-pointer mt-4">`
    - `<input type="checkbox" bind:checked={backedUp} class="w-4 h-4 rounded border-slate-600 bg-slate-700 text-purple-500 focus:ring-purple-500" />`
    - `<span class="text-sm text-slate-300">I've backed up my key</span>`

    **Action buttons (bottom):**
    - Flex row with gap-3, mt-5, border-t border-slate-700, pt-4
    - Cancel button (text, slate-400 hover:white): calls closeModal
    - Logout button (bg-red-600 hover:bg-red-500, disabled when !backedUp):
      - `disabled={!backedUp}`
      - `class="... disabled:opacity-40 disabled:cursor-not-allowed"`
      - calls handleConfirm
      - Text: "Log out"

    Do NOT allow backdrop click or Escape to dismiss without the checkbox — but DO allow it since the user may cancel and return. The modal is re-entrant: canceling is fine, only confirming destroys the key.

         clearAnonymousKey();
         session.set({

           signerType: null,
           displayName: null,
           avatar: null,
           npub: null,
         });

       ```svelte

         npub={$session.npub || ''}
         on:close={() => (showLogoutConfirm = false)}
         on:confirm={doLogout}
       />
       ```
