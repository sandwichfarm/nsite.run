<objective>
Replace the "Update Site" button on the deploy success screen with "Manage sites" and "Deploy another site" buttons, and add an always-visible "Deploy new site" button to the manage view. These component-level changes prepare the navigation options that App.svelte will wire up in Plan 02.

<context>
Phase 15 adds post-action navigation so users are never stuck at a dead end. This plan modifies the two child components (SuccessPanel and ManageSite) to expose the new navigation buttons and events. Plan 02 will handle the App.svelte integration, tab button disabling, and event wiring.

Key decisions from CONTEXT.md:
- D-04: Two buttons on success screen: "Manage sites" and "Deploy another site"
- D-05: These replace the existing "Update Site" button
- D-06: Existing success content (URL, nsec) remains untouched
- D-07: "Manage sites" dispatches 'manage' event; "Deploy another" dispatches 'deploy-another' event
- D-08: Always-visible "Deploy new site" button in manage view content area
- D-09: This is in addition to the Deploy tab button
</context>

<action>
In ManageSite.svelte, add a "Deploy new site" button that is always visible in the manage view content area. This button should appear in multiple states:

**1. In the idle state (deleteState === 'idle'), below the site list (after the `{/each}` closing tag for siteList, around line 313):**

Add after the closing `{/if}` of the siteList block (after line 314):

```svelte
<!-- Deploy new site button -->
<div class="mt-4">
  <button
    on:click={() => dispatch('deploy-new')}
    class="w-full px-4 py-2.5 bg-slate-700 hover:bg-slate-600 text-white rounded-lg text-sm font-medium transition-colors flex items-center justify-center gap-2"
  >
    <svg class="w-4 h-4" fill="none" stroke="currentColor" viewBox="0 0 24 24">
      <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M12 4v16m8-8H4" />
    </svg>
    Deploy new site
  </button>
</div>
```

This button renders both when siteList has items and when it's empty (below the empty state message). It is always enabled in the idle state.

**2. In the delete done state (deleteState === 'done'), alongside the existing "Back to sites" button (around line 590-597):**

Add a second button after the "Back to sites" button:

```svelte
<button
  on:click={() => { handleBackToDeploy(); dispatch('deploy-new'); }}
  class="w-full px-4 py-2.5 bg-slate-700 hover:bg-slate-600 text-white rounded-lg text-sm font-medium transition-colors mt-2"
>
  Deploy new site
</button>
```

Note: This button first resets ManageSite internal state via handleBackToDeploy(), then dispatches 'deploy-new' so App.svelte switches to the deploy tab.

**3. During active deletion (deleteState === 'confirm' or 'deleting'), do NOT show the deploy button** — the user is in the middle of a destructive flow.
</action>
<acceptance_criteria>

- ManageSite.svelte contains text "Deploy new site" in button elements

- The button in done state appears after the "Back to sites" button
- The "Deploy new site" button does NOT appear in the confirm or deleting states (no `dispatch('deploy-new')` inside those template blocks)
- The existing "Back to sites" button (`handleBackToDeploy`) is still present and unchanged
- The existing site list rendering is unchanged (card layout, expand, update, delete buttons all intact)
</acceptance_criteria>
</task>
