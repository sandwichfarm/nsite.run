<objective>
Disable tab buttons during active operations, re-enable them on completion, and wire up the new navigation events from SuccessPanel and ManageSite so post-action navigation works end-to-end.
</objective>

<context>

It adds tab button disabling using the `isDangerousStep` variable from Phase 13, wires up the new events from SuccessPanel ('manage', 'deploy-another') and ManageSite ('deploy-new'), and removes the old "Deploy another site" link that used the wrong reset function.

Phase 13 provides (on enhance/action-guards branch, will be merged before execution):
- `DANGEROUS_DEPLOY_STEPS` Set constant
- `isDangerousStep` reactive derivation (true during hashing/checking/uploading/publishing OR deleteInProgress)
- `deleteInProgress` boolean
- `OperationBanner.svelte` component

CONTEXT.md decisions:
- D-01: Tab buttons disabled (greyed out) during operations, NOT hidden
- D-02: Re-enable on success/error — no waiting for user acknowledgment
- D-03: Same pattern for all operations (deploy, update, delete)
- D-07: "Manage sites" navigates to manage view preserving signer; "Deploy another" uses resetForUpdate()
</context>

      class="flex-1 px-4 py-2 rounded-md text-sm font-medium transition-colors

Key changes:

- When disabled: `opacity-40 cursor-not-allowed text-slate-500` replaces the normal/active styling
- When enabled: existing styling logic preserved exactly (active tab = purple, inactive = slate with hover)

<action>
In App.svelte, update the ManageSite component usage in the idle/selecting section (around lines 559-585) to handle the new 'deploy-new' event.

Current ManageSite usage:
```svelte
<ManageSite
  sites={allSites}
  pubkey={$session.pubkey}
  relayUrls={deleteRelayUrls}
  blossomUrls={deleteBlossomUrls_list}
  signer={currentSigner}
  on:update={(e) => {
    const site = e.detail;
    currentPage = 'deploy';
    if (site.kind === 35128) {
      siteType = 'named';
      dTag = getManifestDTag(site) || '';
      dTagReadOnly = true;
    } else {
      siteType = 'root';
      dTag = '';
      dTagReadOnly = false;
    }
    deployTitle = getManifestTitle(site);
    deployDescription = getManifestDescription(site);
    resetForUpdate();
  }}
  on:deleted={() => {
    fetchSiteInfo($session.pubkey);
  }}
/>
```

Add the new event handler after `on:deleted`:
```svelte
  on:deploy-new={() => {
    currentPage = 'deploy';
    resetForUpdate();
  }}
```

This navigates to the deploy tab and resets the deploy state to the file drop zone while preserving the signer session. `resetForUpdate()` clears file selection, deploy results, and error state but keeps `currentSigner` and `deployNsec` intact. The `currentPage = 'deploy'` assignment happens before `resetForUpdate()` to ensure the tab switches first.

The full ManageSite component usage becomes:
```svelte

  sites={allSites}
  pubkey={$session.pubkey}
  relayUrls={deleteRelayUrls}
  blossomUrls={deleteBlossomUrls_list}

  on:update={(e) => {
    const site = e.detail;
    currentPage = 'deploy';
    if (site.kind === 35128) {
      siteType = 'named';
      dTag = getManifestDTag(site) || '';
      dTagReadOnly = true;
    } else {
      siteType = 'root';
      dTag = '';
      dTagReadOnly = false;
    }
    deployTitle = getManifestTitle(site);
    deployDescription = getManifestDescription(site);

  }}
  on:deleted={() => {
    fetchSiteInfo($session.pubkey);
  }}
  on:deploy-new={() => {
    currentPage = 'deploy';
    resetForUpdate();
  }}
/>
```
