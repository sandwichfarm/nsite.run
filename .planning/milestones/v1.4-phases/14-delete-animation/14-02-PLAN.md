# Plan 14-02: Card exit animation, smooth reflow, and App.svelte integration

<objective>
Add the two-phase CSS exit animation for successfully deleted cards (fade opacity then collapse height), implement smooth reflow of remaining cards, integrate with App.svelte for optimistic card removal, and ensure the success outcome indicator (green flash) displays before the fade-out begins.
</objective>

<context>
After Plan 01, successfully deleted cards have `phase: 'success'` in the `deletingCards` Map but remain in the card list with no exit animation. This plan adds the CSS-only animation sequence: green success flash → fade opacity to 0 → collapse height → remove from list → dispatch deleted event. CSS transitions are used instead of Svelte's `out:fade`/`animate:flip` due to the Svelte 4 bug #4910.
</context>

<task id="2">
<title>Implement the success animation sequence with timed phase transitions</title>
<read_first>
- apps/spa/src/components/ManageSite.svelte (handleConfirmDelete success path from Plan 01, updateCard and removeCard functions)
</read_first>
<action>
After `handleConfirmDelete` sets `phase: 'success'`, add a sequenced animation timer that:

1. **T+0ms:** Card shows "Deleted" label + green success indicator (already done by `phase: 'success'` from Plan 01)
2. **T+600ms:** Add `_collapsing: true` flag to trigger height collapse (the `card-exit-fade` class already started opacity fade at T+0)
3. **T+900ms:** Remove card from siteList and dispatch events

Add a function to orchestrate this:

```javascript
function animateCardExit(siteId) {
  // Phase 1: opacity fade starts immediately via card-exit-fade class (applied by phase === 'success')

  // Phase 2: after fade completes, collapse height
  setTimeout(() => {
    updateCard(siteId, { _collapsing: true });
  }, 600);

  // Phase 3: after collapse, remove from list and notify parent
  setTimeout(() => {
    removeCard(siteId);
    dispatch('site-removed', deletingCards.get(siteId)?.site ?? { id: siteId });
    dispatch('deleted');
  }, 900);
}
```

**Important:** The `dispatch('site-removed', site)` is a NEW event. The `dispatch('deleted')` is the existing event that triggers App.svelte's `fetchSiteInfo`. The new `site-removed` event carries the specific site object so App.svelte can optimistically remove it from `allSites` before the re-fetch completes.

**However**, since `removeCard(siteId)` deletes the Map entry, we need to capture the site reference before the timeout chain. Refactor:

```javascript
function animateCardExit(siteId, site) {
  // Phase 1: opacity fade starts immediately via card-exit-fade CSS class

  // Phase 2: after fade, collapse height
  setTimeout(() => {
    updateCard(siteId, { _collapsing: true });
  }, 600);

  // Phase 3: after collapse, remove and notify
  setTimeout(() => {
    removeCard(siteId);
    dispatch('site-removed', site);
    dispatch('deleted');
  }, 900);
}
```

Call `animateCardExit` from `handleConfirmDelete` right after setting `phase: 'success'`:

In the success branch of `handleConfirmDelete`, after `updateCard(siteId, { phase: 'success', ... })`, add:

```javascript
animateCardExit(siteId, card.site);
```

<task id="3">
<title>Update App.svelte to handle site-removed event for optimistic removal</title>
<read_first>

- apps/spa/src/lib/nostr.js (getManifestDTag function — for identifying named sites)

<action>
Add a `site-removed` event handler on the ManageSite component in App.svelte. This handler optimistically removes the deleted site from `allSites` so the card disappears from the list immediately (the existing `on:deleted` handler still runs `fetchSiteInfo` for relay consistency).

In the ManageSite component element (around line 560), add the new event handler:

```svelte
<ManageSite
  sites={allSites}
  pubkey={$session.pubkey}
  relayUrls={deleteRelayUrls}
  blossomUrls={deleteBlossomUrls_list}
  signer={currentSigner}
  on:update={(e) => {
    const site = e.detail;
    currentPage = 'deploy';
    if (site.kind === 35128) {
      siteType = 'named';
      dTag = getManifestDTag(site) || '';
      dTagReadOnly = true;
    } else {
      siteType = 'root';
      dTag = '';
      dTagReadOnly = false;
    }
    deployTitle = getManifestTitle(site);
    deployDescription = getManifestDescription(site);
    resetForUpdate();
  }}
  on:site-removed={(e) => {
    const site = e.detail;
    if (site.kind === 35128) {
      allSites = {
        ...allSites,
        named: allSites.named.filter(s => s.id !== site.id),
      };
    } else {
      allSites = { ...allSites, root: null };
    }
    existingManifest = allSites.root;
  }}
  on:deleted={() => {
    fetchSiteInfo($session.pubkey);
  }}
/>
```

This ensures:
1. The card is removed from `allSites` immediately when the animation completes (via `site-removed`)
2. The relay re-fetch still happens via `deleted` event for eventual consistency
3. The `existingManifest` is updated for backward compatibility with deploy-side derived values
</action>
